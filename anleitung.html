1 · Überblick

Der Workflow in fünf Schritten:

  1. Web-Tool: PointCheckout-Excel hochladen, Fallback/Alarme/Checkboxen setzen
  2. CSV herunterladen (Niagara-Format, UTF-8, Komma-getrennt)
  3. Robot einmalig als Program-Objekt in der Station einrichten
  4. CSV in den shared-Ordner der Station kopieren
  5. Execute – der Robot legt alle Punkte inkl. Facets, Fallback, Alarm-Extensions und Prioritäts-Flags an
Bereits vorhandene Punkte werden übersprungen – der Robot kann gefahrlos mehrfach laufen.

2 · Web-Tool: Punktliste vorbereiten

  1. PointCheckout-Excel (.xlsx) per Drag&Drop auf die Startseite ziehen
  2. Zähler-Kacheln prüfen: stimmt die Anzahl pro Punkttyp?
  3. Mit den Filtern (Anlage / Typ) die Ansicht eingrenzen
  4. Werte setzen – einzeln in der Tabelle oder via Schnell-Ausfüllen (wirkt auf alle gefilterten Zeilen):
    • fallback – Startwert/Sollwert bei Writables
    • lowAlarm / highAlarm – OutOfRange-Grenzen (nur Numeric)
    • alarm – Checkbox: AlarmSourceExt anhängen ja/nein
    • bacnetExt – Checkbox: Punkt für BACnet-Export vormerken
    • units – Engineering-Einheit (Dropdown)
    • signal (AI) – Fühlertyp: Pt1000, NTC 20kΩ, 0-10V, … (Mehrfachauswahl)
    • NC – bei BI/BO: markiert = Öffner (NC), leer = Schliesser (NO)
  5. «Niagara-CSV herunterladen» (alle Punkte) oder «Auswahl als CSV» (nur Filter)
Spaltenbreiten lassen sich am rechten Rand jeder Überschrift ziehen und werden im Browser gespeichert. Die Daten bleiben komplett lokal – nichts wird hochgeladen.

3 · Robot einrichten (einmalig pro Station)

Schritt 1 – Program anlegen:
Nav-Baum → Config (oder Zielbereich) → Rechtsklick → New → Palette program → Komponente Program → Name: PunktImport
Schritt 2 – Slots anlegen:
PunktImport doppelklicken → Reiter Slots → drei Slots hinzufügen, alle Typ baja:String:
csvPfad · zielOrdner · statusText
⚠️ Exakte Schreibweise – der Code greift per Name darauf zu (csv, nicht cvs!).
Schritt 3 – Imports hinzufügen:
Reiter Imports → Button «Import Package» → nacheinander:
ModulePackage
control-rtjavax.baja.control
alarm-rtjavax.baja.alarm.ext
alarm-rtjavax.baja.alarm.ext.offnormal
bajajavax.baja.naming
bajajavax.baja.units
bajajavax.baja.file
Hinweis: javax.baja.units und javax.baja.file liegen im Modul baja – es gibt kein eigenes units-rt.
Schritt 4 – Code einfügen:
Auf der Startseite «Robot-Code herunterladen» klicken (geht ohne Punktliste) → Datei öffnen → den kompletten Java-Teil (ab public void onExecute()) in den Edit-Tab kopieren → Compile & Save (Hammer-Symbol).
Ziel: grüner Punkt «Program is up-to-date». Das orange «not signed» ist normal und stört nicht.

4 · CSV in die Station bringen

⚠️ Achtung, der häufigste Stolperstein: Es existieren bis zu drei verschiedene «shared»-Ordner – nur einer ist der richtige!
  • ❌ My File System → User Home → shared – Workbench-Ordner, Station sieht ihn nicht
  • ❌ My File System → User Home → stations → <Name> → shared – Offline-Kopie der Station
  • ✅ Das Home der laufenden Station – liegt typischerweise unter C:\ProgramData\Niagara4.15\…\stations\<Name>\

Schritt für Schritt (narrensicher)

  1. csvPfad im Property Sheet auf file:^test.csv setzen (Dateiname anpassen) → Save
  2. Execute ausführen → Refresh → statusText lesen
  3. Es erscheint zunächst absichtlich: CSV nicht gefunden: … | StationHome: C:\ProgramData\…
    → Der Robot verrät dir den exakten Ordner, in dem die Station ihre Dateien sucht
  4. Diesen StationHome-Pfad kopieren und im Windows Explorer in die Adresszeile einfügen
    (ProgramData ist versteckt – Pfad einfach direkt eintippen/einfügen)
  5. Die CSV genau in diesen Ordner kopieren
  6. Execute → Refresh → Neu: 226 | Vorhanden: 0 | Fehler: 0 🎉
Hintergrund: Das ^ in file:^test.csv bedeutet «StationHome der laufenden Station». Je nach Setup endet dieser Pfad bereits auf \shared (z.B. bei lokalen Comfort&Energy-Stationen unter ProgramData) – dann gehört die CSV direkt dorthin und der Pfad ist file:^test.csv ohne weiteres Unterverzeichnis. Auf einem Jace ist das StationHome /opt/niagara/stations/<Name>/ – dort liegt die CSV dann via FileTransferClient im Unterordner shared und der Pfad lautet file:^shared/test.csv. Im Zweifel: Schritt 2–3 oben verrät immer den richtigen Ort.
Die Workbench-Ansicht Station → Files zeigt übrigens genau dieses StationHome – dort siehst du, ob die CSV angekommen ist (Copy/Paste in diese Ansicht funktioniert je nach Build, der Windows-Explorer-Weg ist zuverlässiger).

5 · Import ausführen

  1. zielOrdner exakt ermitteln: Im Nav-Baum auf den Zielordner doppelklicken → in der Adressleiste steht der ORD, z.B. local:|foxs:|station:|slot:/Test_Punkte → nur den Teil ab station: in den Slot übernehmen: station:|slot:/Test_Punkte. Der Ordner muss bereits existieren.
  2. Property Sheet: Save (Diskette)
  3. Rechtsklick auf PunktImport → Actions → Execute
  4. Refresh drücken → statusText lesen:
Neu: 23 | Vorhanden: 0 | Fehler: 0 | BACnet markiert: 5

Danach stichprobenartig prüfen: Einheiten am AI, trueText/falseText am BV, AlarmExt mit Limits am NumericWritable, sichtbare in1/in8/in16 im Slot Sheet (Operator-Flag für EBI/BACnet-Schreibzugriff auf Prio 1/8/16).

BACnet-Export: Die mit bacnetExt markierten Punkte in Workbench multi-selektieren und per Drag&Drop auf Drivers → BacnetNetwork → Local Device → Export Table ziehen.

6 · Fehlerbehebung

MeldungUrsache → Lösung
package javax.baja.control does not exist Import fehlt → Imports-Tab, Tabelle aus Schritt 3 vervollständigen
cannot find symbol: SlotPath / BFormat Alte Code-Version → frischen Robot-Code von dieser Seite laden (SlotPath liegt in javax.baja.naming, BFormat in javax.baja.util)
AccessControlException: access denied FilePermission Direkter C:\-Pfad verwendet → CSV in shared legen, file:^shared/… als csvPfad
UnresolvedException: /Plant/… (zielOrdner) Ordner existiert dort nicht → ORD via Doppelklick + Adressleiste ermitteln
CSV nicht gefunden: … | StationHome: … Kein Fehler, sondern Wegweiser: Den angezeigten StationHome-Pfad im Explorer öffnen und die CSV genau dorthin kopieren (Abschnitt 4). Liegt sie in einem Unterordner, den Unterordner in csvPfad ergänzen: file:^shared/…
Copy/Paste oder Drag&Drop in Workbench-Files schlägt fehl («Clipboard is empty») Workbench-Clipboard-Eigenheit → CSV stattdessen per Windows Explorer direkt ins StationHome kopieren
statusText bleibt leer Refresh-Button drücken; Execute wirklich über Actions ausgelöst?
uses unchecked or unsafe operations Nur eine Warnung – kompiliert trotzdem, ignorieren
«Program is not signed» (orange) Normal bei Program-Objekten – kein Handlungsbedarf

CSV-Spalten

name · displayName · type · bacnetObjectType · units · facets.stateText · terminalAddress · plant · fallback · lowLimit · highLimit · alarmEnable · bacnetExport · signalType · contact

signalType (Pt1000, 0-10V, …) und contact (NO/NC) sind Doku-/IO-Konfigurationsspalten – der Robot ignoriert sie beim Anlegen; sie dienen der späteren Panelbus-Zuordnung.
